# esp32_sorter_screen

## 项目定义
这是一个全新的 ESP32 显示屏端项目（基于原有架构的分支）。该项目主要作为 `esp32_sorter` 主机的 HMI (人机交互界面) 存在。

## 核心特性
- **串口通信**：本屏与主机通过串口连接。
- **职责划分**：
  - 本项目负责：界面显示、触摸交互。
  - 主机 (`esp32_sorter`) 负责：核心业务逻辑、数据处理与下发。
- **通信协议**：废弃 Modbus RTU（由于其二进制特性不适合长报文传输），改用基于 JSON 字符串流的自定义协议（协议名称待定）。
- **极简应用层**：不再需要控制伺服电机、不再负责扫码。原有的 `App` 架构精简，目前几乎只保留 `自检 (Self Test)` 相关的 App。
- **精简驱动层**：移除了 `Belt`、`Modbus Master` 等驱动代码。只保留触摸屏和基础硬件抽象。
